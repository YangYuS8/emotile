## Architecture Checks

### `tickExpression`

- Pure-data function: accepts an expression and explicit tick, returns a new expression.
- Deterministic: same expression and tick produce the same output.
- No runtime loop, timers, `requestAnimationFrame`, browser API, Canvas API, GPU API, or side effects.
- Output remains a normal `EmotileExpression` that validates and can be rendered as pure data.

### Agent Helpers

- `AGENT_GUIDANCE`, `MINIMAL_EXPRESSION`, `COMMON_AGENT_MISTAKES`, and `buildExpression` are local deterministic helpers.
- No model SDK, agent framework, preset expression catalog, or production dependency was introduced.
- Agent-facing input remains validated, normalized, and repairable through the existing schema path.

### Theme/Palette

- `specs/theme-palette-proposal.md` is a design proposal only.
- Runtime implementation is deferred to v0.3 by default unless a concrete downstream integration requires reopening v0.2 scope.
- No Builder runtime issue should be created for theme/palette until that Architect decision changes.
